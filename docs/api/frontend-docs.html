Frontend — Health Access Bridge

Documentación de la SPA React para la plataforma de gestión clínica HAB.

El frontend de HAB es una Single Page Application (SPA) construida con React 18 + TypeScript, que consume la API RESTful del backend FastAPI. Provee interfaces diferenciadas por rol (Admin / Médico) para la gestión clínica de pacientes con discapacidad, análisis predictivo y visualización de métricas.

Ambientes desplegados:
DEV → hab-frontend-dev.onrender.com
QA  → hab-frontend-qa.onrender.com
PROD → hab-frontend.onrender.com

AdminRoute Guard

const AdminRoute = () => {
  const { user } = useAuth();
  if (!user) return <Navigate to="/login" replace />;
  if (user.role !== 'admin') return <Navigate to="/dashboard" replace />;
  return <Outlet />;
};

Contexto de Autenticación

Archivo: src/contexts/AuthContext.tsx

Provee estado global de autenticación. El token JWT y los datos del usuario se persisten en localStorage.

Interfaz AuthContextType

PropiedadTipoDescripción
userUser | nullUsuario autenticado. null si no hay sesión.
login(email, password)Promise<void>Llama a POST /users/login, persiste token, redirige por rol.
logout()voidLimpia token + localStorage + redirige a /login.
isLoadingbooleantrue mientras se verifica el token al inicio.

Interfaz User (interna)

{ id: number; email: string; name: string; role: 'admin' | 'médico' }

Uso

import { useAuth } from '@/contexts/AuthContext';

const { user, login, logout, isLoading } = useAuth();
⚠️ useAuth() lanza un error si se usa fuera de <AuthProvider>.

Contexto de Idioma

Archivo: src/contexts/LanguageContext.tsx

Provee internacionalización (i18n) básica ES/EN. Expone el hook useLanguage() con la función t(key) para traducir cadenas.

ApiService

Archivo: src/services/api.ts · Singleton exportado como apiService.

Gestiona el token JWT en memoria y localStorage. Cubre los endpoints de autenticación y administración de usuarios.

MétodoFirmaEndpoint
login(creds: LoginCredentials) → Promise<TokenResponse>POST /users/login
getCurrentUser() → Promise<User>GET /users/me
registerDoctor(data: RegisterData) → Promise<User>POST /admin/users/register
getUsers(skip?, limit?) → Promise<User[]>GET /admin/users
updateUserStatus(userId, isActive) → Promise<User>PATCH /admin/users/{id}/status
setToken(token: string) → void—
clearToken() → void—

Interfaces

interface LoginCredentials { username: string; password: string; }
interface RegisterData    { email: string; password: string; full_name: string; role: 'admin' | 'médico'; }
interface TokenResponse  { access_token: string; token_type: string; }
interface User           { id: number; email: string; full_name: string; is_active: boolean; role: 'admin' | 'médico'; }

PatientService

Archivo: src/services/patients.ts · Singleton exportado como patientService.

Gestiona el CRUD completo de pacientes y la ejecución del modelo predictivo ML.

MétodoFirmaEndpoint
getPatients(skip?, limit?, search?) → Promise<Patient[]>GET /patients/
getPatient(id: number) → Promise<Patient>GET /patients/{id}
createPatient(data: PatientCreate) → Promise<Patient>POST /patients/
updatePatient(id, data: PatientCreate) → Promise<Patient>PUT /patients/{id}
deletePatient(id: number) → Promise<void>DELETE /patients/{id}
predictPatient(id: number) → Promise<PredictionResult>POST /patients/{id}/predict

Interfaces

interface Patient {
  id: number; nombre_apellidos: string; numero_documento: string;
  fecha_nacimiento: string; edad: number; genero: string;
  orientacion_sexual: string; causa_deficiencia: string;
  cat_fisica: string; cat_psicosocial: string;
  nivel_d1..d6: number; nivel_global: number;
  owner_id: number; prediction_profile: number | null; prediction_description: string | null;
}
interface PredictionResult { profile: number; description: string; }
Si el token expira (HTTP 401), PatientService limpia localStorage y redirige automáticamente a /login.

Página: LoginLanding

Ruta: /login · Archivo: pages/LoginLanding.tsx

Pantalla de acceso al sistema. Incluye formulario de email/contraseña, validación en cliente y llamada a AuthContext.login(). Redirige automáticamente según rol tras el login exitoso.

EventoAcción
Submit formularioLlama login(email, password) del contexto.
Login exitoso — AdminRedirige a /admin
Login exitoso — MédicoRedirige a /dashboard
ErrorMuestra toast de error con el mensaje del backend.

Página: Dashboard

Ruta: /dashboard · Archivo: pages/Dashboard.tsx

Vista principal del médico con métricas clave: total de pacientes, predicciones ejecutadas, distribución de perfiles y accesos rápidos. Usa patientService.getPatients() para calcular estadísticas en cliente.

WidgetFuente de datos
Total pacientesGET /patients/
Predicciones ejecutadasFiltro prediction_profile !== null
Distribución de perfilesConteo local de perfiles 0/1/2
Últimos pacientesSlice de los 5 más recientes

Página: Patients

Ruta: /patients · Archivo: pages/Patients.tsx (~29 KB)

Módulo CRUD completo de pacientes. Incluye tabla con búsqueda debounced, diálogo de creación/edición, confirmación de eliminación y exportación a Excel/PDF.

FuncionalidadDetalle
ListarpatientService.getPatients(skip, limit, search)
CrearFormulario en diálogo modal → createPatient()
EditarFormulario pre-relleno → updatePatient()
EliminarConfirm dialog → deletePatient()

ExportarCliente: Excel (SheetJS) / PDF (jsPDF)

Página: Predictions

Ruta: /predictions · Archivo: pages/Predictions.tsx

Módulo de predicción ML. Permite seleccionar un paciente, ejecutar el modelo HybridModelDisability y ver el historial de predicciones con badges de color por perfil.

PerfilBadgeDescripción
0Barreras BajasPaciente con mínimas barreras de acceso.
1Barreras ModeradasRequiere intervención preventiva.
2Barreras AltasPrioridad de atención e intervención inmediata.

Página: Analytics

Ruta: /analytics · Archivo: pages/Analytics.tsx

Dashboard analítico con gráficas Recharts. Muestra distribución de perfiles predictivos (gráfica de torta), métricas demográficas y tablas de resumen.

Componentes Recharts usados

PieChart · BarChart · ResponsiveContainer · Tooltip · Legend

Página: PredictiveGuide

Ruta: /predictive-guide · Archivo: pages/PredictiveGuide.tsx

Guía clínica estática que explica la interpretación de los 3 perfiles ICF del modelo predictivo. Incluye criterios de clasificación, recomendaciones de intervención y glosario.

Página: AdminPanel

Ruta: /admin · Archivo: pages/AdminPanel.tsx (~15 KB) · Solo Admin

Panel de administración con gestión completa de usuarios: listar, crear nuevos médicos/admins, activar/desactivar cuentas y visualizar estadísticas globales del sistema.

DashboardLayout
