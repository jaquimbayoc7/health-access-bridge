Health Access Bridge  [Software System]
Frontend SPA
[Container: React 18 / TypeScript / Vite / TailwindCSS]
App de página única. Provee UI para login (RBAC), gestión de pacientes, predicciones de discapacidad y panel de administración. Build: npm run build → dist/.
Backend API
[Container: Python 3.11 / FastAPI / SQLAlchemy / Uvicorn]
API REST con autenticación JWT y RBAC (Admin / Médico). Gestiona pacientes (CRUD), delega predicciones al HybridModelDisability API externo y expone /docs (Swagger UI). Puerto $PORT vía Uvicorn.
PostgreSQL 15
[Container: PostgreSQL — Render Managed DB]
Almacena usuarios, pacientes, historias clínicas y predicciones. 3 instancias aisladas: hab-db-dev / hab-db-qa / hab-db-prod (Ohio).
HybridModelDisability API
[Software System — ML Service]
Servicio externo alojado en Render.
URL: hybridmodeldisability.onrender.com
Predice perfiles de barreras de discapacidad (K-Means + Gradient Boosting) basados en niveles de actividad D1–D6.
GitHub Actions
[Software System — CI/CD]
Workflows automatizados (ci-dev.yml / ci-prod.yml). Ejecuta pytest + build Vite en 3 ambientes (develop → dev, staging → qa, master → prod). Gate de aprobación manual en producción.
Render Cloud Platform
[Software System — PaaS / Blueprint]
Hospeda Backend (Web Service Python), Frontend (Static Site) y PostgreSQL Managed en 3 ambientes independientes, definidos en render.yaml (Blueprint). Región: Ohio.
