Contexto y Problema

¿Por qué existe Health Access Bridge?

🚧

El Problema

Las personas con discapacidad en zonas rurales de Colombia enfrentan barreras de acceso al sistema de salud que no son identificadas ni sistematizadas eficientemente.

💡

La Solución

Plataforma web con IA que permite a médicos registrar pacientes, ejecutar predicciones de perfiles de barreras (Baja/Moderada/Alta) y consultar guías de intervención.

📊

El Impacto

Modelo HybridModelDisability (K-Means + Gradient Boosting) clasifica perfiles usando dimensiones CIF D1-D6 para priorización clínica en campo.

HU-11 Smoke Tests en Producción ci-prod.yml · 2 escenarios
✅ 2/2▶
 
Feature: Smoke Tests de Producción

  Background:
    Given el pipeline ejecutó un despliegue exitoso en Render producción
    And han transcurrido al menos 90 segundos desde el inicio del contenedor

  Scenario: TC-S01 — Health check responde HTTP 200
    Given el endpoint GET /health está expuesto
    When  el job smoke test realiza GET /health
    Then  la respuesta tiene código 200
    And   el servicio se considera operativo

  Scenario: TC-S02 — Endpoint de login es alcanzable
    Given el endpoint POST /users/login está expuesto
    When  el job smoke test realiza POST /users/login sin credenciales
    Then  la respuesta tiene código 200, 401 o 422 (no 5xx)
    And   el pipeline continúa si todos los checks pasan

HU-13 Frontend — AuthContext AuthContext.test.tsx · 4 escenarios
✅ 4/4▶
 
Feature: Contexto de Autenticación (AuthContext)

  Background:
    Given AuthProvider envuelve la app en MemoryRouter
    And   apiService está mockeado (login, getCurrentUser, setToken, clearToken)
    And   localStorage está limpio antes de cada test

  Scenario: TC-F01 — Estado inicial sin sesión activa
    Given no existe token ni usuario en localStorage
    When  se renderiza useAuth() y isLoading resuelve a false
    Then  user debe ser null

  Scenario: TC-F02 — Login almacena usuario en estado y localStorage
    Given apiService.login responde {access_token: "test-token"}
    And   apiService.getCurrentUser responde {email: "admin@hab.co", role: "admin"}
    When  se llama login("admin@hab.co", "adminpassword")
    Then  user.email = "admin@hab.co"
    And   localStorage contiene la clave "user"

  Scenario: TC-F03 — Logout limpia estado y localStorage
    Given localStorage tiene authToken y user de sesión activa
    When  se llama logout()
    Then  user = null
    And   localStorage no contiene authToken ni user

  Scenario: TC-F04 — Credenciales inválidas no actualizan estado
    Given apiService.login lanza Error("Unauthorized")
    When  se llama login("bad@email.com", "wrongpassword")
    Then  la función lanza excepción
    And   user permanece null
